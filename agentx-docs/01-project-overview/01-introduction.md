# 项目简介

## 什么是 AgentX？

AgentX 是一个基于 Java Spring Boot 构建的**企业级 AI 智能体平台**，专为金融、政务等垂直场景深度优化。它不仅提供通用的 AI Agent 能力，更通过预置行业工具、工作流模板和安全合规机制，帮助企业快速构建符合业务需求的智能应用。

> 💡 **核心定位**：AgentX 不是"另一个 AI 框架"，而是"**能解决企业实际问题**"的生产级解决方案。

## 诞生背景

在 AI 技术爆发的 2026 年，企业面临三大困境：

1. **技术门槛高**：90% 的 AI 教程基于 Python，Java 开发者面临技术栈割裂
2. **部署成本高**：私有化部署动辄需要高端 GPU 服务器，中小企业难以承受
3. **落地效果差**：多数 Demo 停留在"玩具级"，缺乏真实业务场景的闭环验证

AgentX 正是为解决这些问题而生：

- **技术栈统一**：基于 Java 生态，无缝集成企业现有系统
- **低配服务器支持**：2 核 4G 内存即可运行完整系统
- **业务场景闭环**：内置金融风控、政务流程等真实业务工具

## 设计理念

### 1. 垂直领域优先

AgentX 拒绝"通用但平庸"的设计思路，专注于金融、政务等特定行业：

| 通用 AI 框架 | AgentX（垂直优化） |
|--------------|-------------------|
| 提供基础 AI 能力 | **预置行业工具**（风控规则查询、政务流程导航） |
| 需要大量定制开发 | **开箱即用**的业务工作流模板 |
| 安全合规需自行实现 | **内置金融级**安全控制和合规检查 |

### 2. 低配服务器友好

AgentX 的架构设计充分考虑中小企业资源限制：

- **内存优化**：智能内存管理，4GB 内存可同时运行 Spring Boot、Milvus、Redis
- **CPU 优化**：支持 CPU 量化模型，无需 GPU 加速
- **响应优化**：Redis 缓存 + 异步处理，提升并发能力

### 3. 工程化落地

企业级应用的核心是稳定性和可维护性：

- **强类型安全**：Java 类型系统减少运行时错误
- **完整可观测性**：OpenTelemetry 分布式追踪 + Prometheus 监控
- **标准化协议**：MCP（Model Context Protocol）确保工具生态互通

## 与现有系统的关系

AgentX 设计为"增强型"而非"替代型"系统：

### 集成现有业务系统
- **数据层**：通过 JDBC/API 连接企业现有数据库
- **服务层**：通过 REST/WebSocket 调用内部微服务
- **安全层**：集成企业现有的认证授权系统

### 补充 AI 能力
- **智能决策**：在原有业务规则基础上增加 AI 推理
- **自然交互**：为传统系统添加对话式交互界面
- **知识增强**：通过向量检索增强现有知识库

### 渐进式演进
- **阶段 1**：作为独立服务提供 AI 能力
- **阶段 2**：深度集成核心业务流程
- **阶段 3**：逐步替代部分传统业务模块
