

# 🚀 全栈开发者高效技术实务指南

## 第一部分：开发环境优化与系统配置

### 1. IntelliJ IDEA 高效注释与版权配置
统一的代码规范是团队协作的基础，通过 IDEA 的模板功能可极大提升开发效率。

#### 1.1 新建文件头注释 (File Header)
*   **路径**：`Settings` -> `Editor` -> `File and Code Templates` -> `Includes` -> `File Header`
*   **推荐模板**：
    ```java
    /**
     * @description TODO
     * @author ${USER}
     * @date ${DATE} ${TIME}
     * @version 1.0
     */
    ```

#### 1.2 方法注释模板 (Live Templates)
实现输入 `/**` + `Enter` 自动提取参数与返回值。
1.  **路径**：`Settings` -> `Editor` -> `Live Templates`。
2.  **创建**：新建 Template Group（如 `MyCustom`），在其下新建 Live Template。
3.  **配置**：
    *   **Abbreviation**: `*`
    *   **Template text**:
        ```java
        *
         * @description $description$
         * @author $user$
         * @date $date$ $time$
         $params$
         * @return $return$
         */
        ```
    *   **Variables 配置**：
        *   `user`: `user()`
        *   `date`: `date()`
        *   `time`: `time()`
        *   `return`: `methodReturnType()`
        *   **`params` (关键脚本)**：复制以下代码至 Expression 栏：
          `groovyScript("def result=''; def params=\"${_1}\".replaceAll('[\\\\[|\\\\]|\\\\s]', '').split(',').toList(); for(i = 0; i < params.size(); i++) {if(params[i] == '') return result;if(i==0) result += '* @param ' + params[i] + ((i < params.size() - 1) ? '\\n' : '');else result += ' * @param ' + params[i] + ((i < params.size() - 1) ? '\\n' : '')}; return result", methodParameters())`

#### 1.3 类注释与版权 (Copyright)
*   **既有类注释快捷键**：在 `Live Templates` 设置缩写 `cc`，内容同文件头注释，适用范围勾选 `Java -> Declaration`。
*   **版权声明**：在 `Settings -> Editor -> Copyright -> Copyright Profiles` 新建模板，并在 `Copyright` 主页面将 `Default project copyright` 设置为该模板。

### 2. 系统用户名与开发路径修复
**警告：** 若 Windows 用户名为中文（如 `C:\Users\张三`），常导致 Maven、Gradle 或 Python 环境崩溃。

*   **安全修改（仅显示名称）**：通过 Microsoft 账号官网或控制面板修改昵称。
*   **彻底修复（开发者推荐）**：
    1.  **新建英文管理员账号**：设置 -> 账户 -> 其他用户 -> 添加没有 Microsoft 账户的用户。
    2.  **赋权**：将账户类型更改为“管理员”。
    3.  **迁移**：登录新账号，将旧路径下的文档、桌面文件拷贝至新路径（如 `C:\Users\dev`），在新环境下重新配置开发环境。

## 第二部分：Git 与 GitHub 深度管理

### 1. 基础提交标准流程
```bash
git status               # 查看改动
git add .                # 加入暂存区
git commit -m "描述说明"  # 提交到本地仓库
git push origin main     # 推送到远程分支
```
*   **身份验证**：推荐配置 **SSH Key** 免密提交；HTTPS 方式需使用 **PAT (Token)**。

### 2. 解决 “Your local changes will be overwritten by merge” 错误
当远程更新与本地未提交代码冲突时，根据需求选择：

| 场景 | 推荐操作 | 核心命令 |
| :--- | :--- | :--- |
| **暂存改动，稍后继续** | **方案 A (Stash)** | `git stash` -> `git pull` -> `git stash pop` |
| **完成开发，准备合并** | **方案 B (Commit)** | `git add .` -> `git commit` -> `git pull` |
| **放弃本地，以远程为准** | **方案 C (Discard)** | `git reset --hard HEAD` -> `git pull` |

### 3. GitHub 仓库清理与废弃管理
*   **网页端删除**：仓库 `Settings` -> 滚动到底部 `Danger Zone` -> `Delete this repository` -> 输入 `用户名/仓库名`。
*   **命令行删除**：`gh repo delete 用户名/仓库名 --confirm`。
*   **替代方案**：若想保留代码但不维护，选择 **Archive this repository**（归档），仓库将变为只读。

## 第三部分：特定项目实战：Spring AI 配置排查

针对 Spring AI 示例项目 (`spring-projects/spring-ai-examples`) 的配置要点：

### 1. 依赖报错 `Dependency not found` 解决
通常由于使用了不存在的超前版本或未配置仓库：
*   **修正版本**：建议 Java 使用 `17` 或 `21`；Spring Boot 使用 `3.4.2`；Spring AI 使用 `1.0.0-M5`。
*   **配置仓库 (Maven)**：必须添加 Spring Milestone 仓库，否则无法下载 M 系列版本。
    ```xml
    <repositories>
        <repository>
            <id>spring-milestones</id>
            <url>https://repo.spring.io/milestone</url>
        </repository>
    </repositories>
    ```

### 2. 环境要求
*   **运行前提**：必须配置环境变量 `OPENAI_API_KEY`。
*   **构建指令**：进入项目根目录执行 `./mvnw clean install`。

## 第四部分：开发者高效工具选型

### 1. PDF 处理 (免费且强大)
*   **PDFgear**：完全免费，支持像 Word 一样直接修改 PDF 文字，集成 AI。
*   **PDF24 Creator**：离线处理，适合对隐私敏感的办公文档。

### 2. 逻辑绘图与架构
*   **Draw.io (diagrams.net)**：专业级、开源，支持 GitHub 存储。
*   **Mermaid**：适合在 Markdown 中通过代码直接生成流程图。
*   **Excalidraw**：手绘风格，适合头脑风暴。

### 3. Markdown 写作与知识库
*   **MarkText**：Typora 的最佳免费开源替代品，所见即所得。
*   **Obsidian**：基于双向链接的本地知识库，适合构建“第二大脑”。
*   **VS Code**：配合 `Markdown All in One` 插件，适合边写代码边记录文档。

## 💡 总结建议
1.  **习惯养成**：勤提交 (`commit`)，先拉取 (`pull`) 后开发，避免大规模冲突。
2.  **环境至上**：优先解决系统路径中文问题，避免后续各类工具链的玄学报错。
3.  **文档规范**：利用 IDEA 模板强制生成注释，为代码审计和团队交接打好基础。
