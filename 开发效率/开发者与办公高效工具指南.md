## 一、 PDF 处理工具推荐
如果你需要处理 PDF 文档，以下是针对不同需求的最佳免费方案：

| 工具名称 | 核心优势 | 推荐场景 |
| :--- | :--- | :--- |
| **PDFgear** | **完全免费且能直接修改原文**，集成 AI 助手 | 需要像编辑 Word 一样改写 PDF 内容 |
| **PDF24 Creator** | 纯离线运行，功能极其丰富（合并、压缩、OCR） | 批量处理敏感办公文件，注重隐私 |

## 二、 GitHub 代码仓拉取 (以 Spring AI 为例)
针对 `spring-projects/spring-ai-examples` 项目的操作要点：

1.  **基础指令**：
    ```bash
    git clone https://github.com/spring-projects/spring-ai-examples.git
    ```
2.  **环境要求**：
    *   **JDK**: 必须 17 或更高版本。
    *   **环境变量**: 运行 OpenAI 示例需配置 `OPENAI_API_KEY`。
3.  **快速运行**：
    使用项目自带的 Maven Wrapper：`./mvnw clean install`。

## 三、 流程图与逻辑绘图工具
绘制业务流程或系统架构时的首选：

*   **专业级 (Visio 替代)**: **Draw.io (diagrams.net)**
    *   完全开源免费，支持导出多种格式，可关联 GitHub/OneDrive 存储。
*   **极速草图**: **Excalidraw**
    *   手绘风格，适合头脑风暴和快速演示。
*   **代码绘图**: **Mermaid**
    *   直接在 Markdown 中通过文本生成图表，开发者友好。
*   **国内协作**: **ProcessOn**
    *   中文支持好，拥有丰富的中文模板库（免费版限 9 张图）。

## 四、 Markdown 编辑与笔记工具
根据写作习惯选择最适合的编辑器：

1.  **所见即所得 (Typora 免费替代)**: **MarkText**
    *   开源、简洁，输入即渲染，体验极佳。
2.  **知识库管理 (第二大脑)**: **Obsidian**
    *   支持双向链接、插件丰富，适合建立个人知识体系。
3.  **程序员全能王**: **VS Code**
    *   安装 `Markdown All in One` 插件，实现边写代码边写文档。
4.  **跨端同步**: **Joplin**
    *   开源，支持配合坚果云/OneDrive 实现手机与电脑同步。

## 五、 Git 代码提交标准流程
在本地修改代码后，提交到 GitHub 的标准步骤：

### 1. 基础提交步骤
```bash
git status               # 查看改动
git add .                # 将所有改动加入暂存区
git commit -m "描述说明"  # 提交到本地仓库
git push origin main     # 推送到远程分支
```

### 2. 身份验证提醒
*   **SSH Key**: 推荐配置 SSH 密钥，实现免密提交。
*   **PAT (Token)**: 如果使用 HTTPS，需在 GitHub 生成 Token 代替账号密码。

### 3. 向开源项目贡献 (PR) 流程
1.  在 GitHub 页面点击 **Fork** 到个人名下。
2.  `git clone` 你的个人副本仓库。
3.  修改并 `push` 到你的仓库。
4.  在 GitHub 提交 **Pull Request** 等待原作者审核。
