
# 开发者常见问题处理手册

## 第一部分：GitHub 废弃仓库管理
在项目迭代过程中，清理不再维护的废弃仓库是保持工作区整洁的重要步骤。

### 1. 通过网页端删除（最常用）
1. 登录 GitHub，进入目标仓库主页。
2. 点击顶部菜单栏最右侧的 **Settings**。
3. 滚动到页面最底部的 **Danger Zone**（危险区域）。
4. 点击 **Delete this repository**。
5. 按照提示点击确认，并在文本框中输入 `用户名/仓库名` 以确认删除。

### 2. 通过 GitHub CLI 命令行删除（最高效）
如果您安装了 GitHub CLI，可以直接在终端执行：
```bash
gh repo delete 用户名/仓库名 --confirm
```

### 3. 注意事项
*   **不可逆性**：删除操作是永久性的，无法轻易撤销。
*   **归档建议**：如果您想保留代码但不再维护，建议选择 **Archive this repository**（归档）。归档后仓库变为“只读”，且不会在活跃列表中显示。
*   **权限影响**：删除私有仓库会同步删除该仓库的所有分叉（Forks）。

## 第二部分：Spring AI 项目依赖报错排查
针对 Maven 报错 `Dependency 'org.springframework.ai:spring-ai-starter:' not found` 的分析与解决方案。

### 1. 错误原因分析
报错的核心在于 `pom.xml` 中使用了**尚未发布或不存在的版本号**，导致 Maven 无法从远程仓库获取依赖元数据（BOM）：
*   **版本超前**：Spring Boot `4.0.3`、Java `25` 以及 Spring AI `2.0.0` 在当前（2025年初）均属于不存在的版本。
*   **依赖命名更迭**：Spring AI 的 Starter 命名在不同里程碑版本间有变动。

### 2. 推荐的稳定配置 (pom.xml)
建议将版本降级至当前可用的里程碑版本（Milestone）：

```xml
<properties>
    <!-- 使用当前主流的 LTS Java 版本 -->
    <java.version>21</java.version>
    <!-- 使用当前可用的 Spring AI 里程碑版本 -->
    <spring-ai.version>1.0.0-M5</spring-ai.version>
</properties>

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <!-- 建议使用 3.4.x 系列稳定版 -->
    <version>3.4.2</version>
</parent>

<dependencies>
    <!-- Spring Web 核心依赖 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- OpenAI 专用 Starter (最新规范命名) -->
    <dependency>
        <groupId>org.springframework.ai</groupId>
        <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
    </dependency>
</dependencies>

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.ai</groupId>
            <artifactId>spring-ai-bom</artifactId>
            <version>${spring-ai.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<repositories>
    <!-- 必须配置 Milestone 仓库才能下载 M5 等版本 -->
    <repository>
        <id>spring-milestones</id>
        <name>Spring Milestones</name>
        <url>https://repo.spring.io/milestone</url>
    </repository>
</repositories>
```

### 3. 解决步骤建议
1.  **修正版本号**：将 Java 改为 `17` 或 `21`；Spring Boot 改为 `3.4.2`；Spring AI 改为 `1.0.0-M5`。
2.  **更新依赖名**：将 `spring-ai-starter-model-openai` 更改为官方推荐的 `spring-ai-openai-spring-boot-starter`。
3.  **刷新 Maven 缓存**：
    *   在 IDE 中执行 **Reload All Maven Projects**。
    *   或在命令行执行：`mvn clean install -U`。
