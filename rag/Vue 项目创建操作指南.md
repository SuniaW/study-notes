


# Vue 项目创建操作指南

## 📝 概述
目前前端生态中，创建 Vue 项目主要分为两种主流构建工具。本指南将指导您完成从环境准备到项目启动的全过程。
1. **基于 Vite**：新一代构建工具，极速冷启动和热更新，是 **Vue 官方目前首推** 的创建方式（默认创建 Vue 3 项目）。
2. **基于 Vue CLI**：基于 Webpack，功能全面但构建速度稍慢。目前处于维护模式，主要用于维护老项目或公司强制要求 Webpack 的场景。

---

## 🛠️ 一、 环境准备 (必备)

无论是哪种创建方式，都需要依赖 Node.js 环境。

1. **检查是否已安装**：
   打开终端（cmd / PowerShell / Terminal），输入以下命令检查版本：
   ```bash
   node -v
   ```
2. **下载与安装**：
   如果未安装或版本过低（建议 Node.js >= 18.0.0），请前往 [Node.js 官网](https://nodejs.org/) 下载并安装 **LTS（长期支持）版本**。

3. **配置国内镜像（可选，针对网络较慢的情况）**：
   ```bash
   npm config set registry https://registry.npmmirror.com
   ```

---

## 🚀 二、 使用 Vite 创建项目（🌟 官方推荐）

如果您正在开启一个全新的项目，请毫不犹豫地选择此方式。

### 1. 执行创建命令
在终端中进入您想要存放项目的目录，运行以下命令：
```bash
npm create vue@latest
```

### 2. 填写配置信息
命令执行后，终端会启动一个交互式的向导，您可根据实际需求通过键盘选择（如果不确定，除了项目名称外，其余可直接按回车键选 `No`）：

| 提示信息 (Prompt) | 说明 | 新手建议 |
| :--- | :--- | :--- |
| **Project name** | 项目名称（如 `my-vue-app`） | 自定义 |
| **Add TypeScript?** | 是否引入 TypeScript 支持 | `No` |
| **Add JSX Support?** | 是否引入 JSX 语法支持 | `No` |
| **Add Vue Router?** | 是否引入路由（单页应用/多页面必备）| 视需求定 |
| **Add Pinia?** | 是否引入状态管理（替代原 Vuex） | 复杂项目选 `Yes` |
| **Add Vitest?** | 是否引入单元测试框架 | `No` |
| **Add ESLint?** | 是否引入代码质量检查工具 | `Yes` |
| **Add Prettier?** | 是否引入代码格式化工具 | `Yes` |

### 3. 安装依赖并启动
项目创建完成后，终端会给出操作提示，依次执行以下命令：
```bash
cd <你的项目名称>      # 1. 进入项目根目录
npm install           # 2. 安装项目所需依赖
npm run dev           # 3. 启动本地开发服务器
```
启动成功后，浏览器访问控制台输出的地址（通常为 `http://localhost:5173`）即可预览项目。

## 📦 三、 使用 Vue CLI 创建项目（维护模式）

适用于需要兼容旧版 Vue 2 或深度依赖 Webpack 生态的项目。

### 1. 全局安装 Vue CLI 工具
*注：Mac 系统如果提示权限不足，请在命令前添加 `sudo`。*
```bash
npm install -g @vue/cli
```

### 2. 执行创建命令
```bash
vue create <你的项目名称>
```

### 3. 选择项目预设
通过键盘上下键选择需要的预设方案：
* `Default ([Vue 3] babel, eslint)`：一键创建默认配置的 Vue 3 项目。
* `Default ([Vue 2] babel, eslint)`：一键创建默认配置的 Vue 2 项目。
* `Manually select features`：手动选择特性（可按需勾选 Router、Vuex、CSS 预处理器等）。

### 4. 启动项目
```bash
cd <你的项目名称>
npm run serve
```

## 🖥️ 四、 使用图形化界面创建（Vue UI）

如果您更习惯通过可视化界面操作，Vue CLI 提供了一个基于网页的管理控制台。

1. 确保已按“三、1”中的步骤全局安装了 Vue CLI。
2. 在任意终端位置输入并执行：
   ```bash
   vue ui
   ```
3. 浏览器会自动打开 `http://localhost:8000`（或相似端口）。
4. 在页面顶部选择“**创建**”标签页，选择项目存放路径。
5. 点击“在此创建新项目”，根据页面提示输入名称并勾选所需插件，点击创建即可。

## 💡 五、 开发者最佳实践建议

为了获得最佳的开发体验，建议进行以下工具链配置：

1. **开发编辑器**：强烈推荐使用 **Visual Studio Code (VS Code)**。
2. **核心插件配置**：
    * 针对 **Vue 3 (Vite)** 项目：请在 VS Code 中搜索并安装 `Vue - Official` 插件（原名 Volar）。
    * 针对 **Vue 2 (CLI)** 项目：推荐使用 `Vetur` 插件。
    * ⚠️ **注意**：`Vue - Official` 和 `Vetur` 会产生冲突，请务必根据当前开发的项目版本，**禁用掉另一个插件**。

> **结论**：对于绝大多数从零开始的现代 Web 应用，请始终首选 **方式二（Vite）** 进行初始化。
