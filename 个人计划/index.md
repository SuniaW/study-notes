### 阶段一：从“单体 RAG”向“Agentic Workflow（智能体工作流）”跃迁
**目标**：让你的系统从“被动查资料的只读系统”变成“主动执行任务的操作员”。
**预计耗时**：3 周左右（每天按 2-3 小时算）

*   **Week 1：彻底打通 Function Calling (工具调用)**
    *   **学习内容**：深挖 Spring AI 中的 `@Tool` 或 `FunctionCallback` 机制。理解大模型是如何输出 JSON 来触发本地 Java 方法的。
    *   **实战任务**：给你的 `Enterprise-RAG` 增加几个本地工具类。
        1. `WeatherService`: 调用高德/心知天气API查询天气。
        2. `DatabaseQueryTool`: 让大模型通过自然语言直接查询你本地的 MySQL 数据库（比如统计当天的注册用户数）。
    *   **产出物**：你的问答系统现在能回答：“今天西安天气如何？顺便帮我查一下昨天系统新增了多少工单？”

*   **Week 2：探索前沿的 Agent 框架（Python为主，开拓视野）**
    *   **学习内容**：虽然你是 Java 主力，但目前最牛的 Agent 思想都在 Python 圈。去学习 **LangGraph** 或 **CrewAI** 的基础概念。理解什么是“状态机(State Graph)”、“多智能体协作”。
    *   **实战任务**：用 Python 写一个简单的脚本。定义两个 Agent：一个是“资料搜集员”，一个是“报告撰写员”，让它们配合完成一篇特定主题的短文。

*   **Week 3：在业务中落地 Agentic Workflow**
    *   **实战任务**：在你的政务问答系统中，加入“业务办理”流程。
    *   **场景设计**：用户问“我要怎么申请生育津贴？” -> 系统触发 RAG 查询政策 -> 接着系统主动问“你需要我现在帮你填写申请草稿吗？” -> 用户同意 -> 系统触发 `FormSubmitTool` 模拟提交数据。
    *   **产出物**：一个具备“多轮对话 + 查资料 + 填表单”能力的完整政务 Agent 雏形。

### 阶段二：攻克企业级 AI 深水区（图谱与评测）
**目标**：解决目前大模型落地的最大痛点——“跨文档复杂推理能力差”和“如何向老板证明 AI 没瞎编”。
**预计耗时**：4 周左右

*   **Week 1-2：引入图数据库，实战 GraphRAG**
    *   **背景优势**：你之前履历中写过熟练掌握 **Neo4j**，这在现在是极大的加分项！
    *   **学习内容**：了解 Knowledge Graph (知识图谱) + RAG 的结合方案。了解如何用 LLM 把一段文本抽取成 `实体-关系-实体`（三元组）。
    *   **实战任务**：在 Spring Boot 中集成 Neo4j。拿几篇复杂的政务审批流程文档，让 LLM 自动抽取其中的部门关系、审批流转关系存入图数据库。
    *   **产出物**：实现一次 Cypher 查询 + 向量检索的“混合查询”，大幅提升复杂逻辑问题的准确度。

*   **Week 3：攻克复杂文档解析（PDF/表格）**
    *   **痛点**：你目前用的是 Apache Tika，处理纯文本很好，但遇到带表格的 PDF 效果很差。
    *   **学习/实战**：研究并接入更强的文档解析引擎。比如开源的 **Marker**，或者阿里云文档解析API。重点解决“文档中表格数据的精确提取与分块(Chunking)”。

*   **Week 4：建立量化评测体系 (LLM Evaluation)**
    *   **学习内容**：学习开源评测框架 **Ragas**（Python）。理解评估 RAG 的三大核心指标：上下文精确度(Context Precision)、忠实度(Faithfulness)、答案相关性(Answer Relevance)。
    *   **实战任务**：写一套 Python 自动化测试脚本，准备 50 个典型的测试问题和标准答案。跑一遍你的系统，生成一份评测报告（比如：准确率 92%，幻觉率 3%）。
    *   **产出物**：这套“评测报告”是你对外接单、面试高级岗位的绝对杀手锏，证明你做的是“工业级”而不是“玩具”。

### 阶段三：轻量级模型微调 (SFT)（开拓边界）
**目标**：懂微调的边界，能在需要特定语气、特定 JSON 格式输出时，提供本地微调方案。
**预计耗时**：2 周左右（此阶段偏向“跑通流程”）

*   **Week 1：环境搭建与数据准备**
    *   **工具**：不要在本地电脑配环境，去 **AutoDL** 租一台带 RTX 3090 或 4090 的云服务器（每小时大约 2 块钱）。
    *   **框架**：学习使用 **LLaMA-Factory**（一个非常好用的开源微调框架，带 Web 界面）。
    *   **实战任务**：准备 100-200 条专属 QA 数据（比如带有特定政务黑话、或特定 JSON 格式的回复）。

*   **Week 2：LoRA 微调与部署**
    *   **实战任务**：在 LLaMA-Factory 中加载 Qwen-1.5B 或 7B 模型，使用准备好的数据进行 LoRA 监督微调 (SFT)。
    *   **导出**：将微调后的模型权重导出，转换为 GGUF 格式。
    *   **验证**：把这个你“自己炼”的模型，挂载到你本地的 Ollama 里，测试它是否学会了你教它的特定语气。
